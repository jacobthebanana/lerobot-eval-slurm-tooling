1. Store large files (model weights, datasets) under $SCRATCH.
2. Use astral-uv `uv` for virtual environment, and use $SCRATCH/uv-venv/(folder name) for the virtual environment. Important: do not create virtual environments in the home directory (including this folder)- only in $SCRATCH.
3. Run experiments in parallel by submitting them to the `long` partition of the SLURM cluster - see mila-docs for details. Be sure to set monitors and timers (to identify runs that are stuck) to keep track of the status of the jobs. Use `unkillable` for short test runs only (this SLURM partition is limited to 1 concurrent GPU at a time).
